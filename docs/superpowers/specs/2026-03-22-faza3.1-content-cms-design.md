# Faza 3.1 — Content & CMS: Blog + Pagini legale + Cookie consent

## Scope

Three independent subsystems delivered together:

1. **Payload CMS** (`apps/cms`) — headless CMS for blog posts and legal/static pages
2. **Storefront content pages** — `/blog`, `/blog/[slug]`, `/blog/categorii/[slug]`, `/pagini/[slug]`, navigation updates
3. **Cookie consent banner** — full GDPR-compliant banner (bar bottom, categories)

---

## Section 1: Payload CMS (`apps/cms`)

### Architecture

Payload CMS runs as a separate app in the monorepo at `apps/cms`, port **3001**. It connects to the same PostgreSQL instance as Medusa but uses its own table prefix (`cms_`). Turborepo orchestrates all three apps via `pnpm dev`.

### Collections

**`posts`** — blog articles

| Field | Type | Notes |
|---|---|---|
| `title` | text | required |
| `slug` | text | required, unique, auto-generated from title |
| `category` | relationship → `categories` | required |
| `coverImage` | upload | required |
| `excerpt` | textarea | max 200 chars |
| `content` | rich text (Lexical) | main article body |
| `author` | text | |
| `publishedAt` | date | |
| `status` | select: `draft` \| `published` | default: `draft` |

**`pages`** — legal + static pages (About, GDPR, T&C, Returns)

| Field | Type | Notes |
|---|---|---|
| `title` | text | required |
| `slug` | text | required, unique (e.g., `despre-noi`, `gdpr`, `termeni-si-conditii`, `retur`) |
| `content` | rich text (Lexical) | |
| `showInFooter` | checkbox | controls footer nav visibility |

**`categories`** — blog categories

| Field | Type | Notes |
|---|---|---|
| `name` | text | required |
| `slug` | text | required, unique (e.g., `ghiduri`, `noutati`, `produse-in-focus`) |

### Admin UI

Payload admin at `http://localhost:3001/admin`. Credentials separate from Medusa admin.

### File structure

```

├── payload.config.ts         ← Payload configuration
├── src/
│   ├── collections/

│   │   ├── Pages.ts

```

## Section 2: Storefront — Content Pages

### URL Structure

| URL | Content | revalidate |
|---|---|---|
| `/blog` | All posts listing, client-side category filter | 3600 |
| `/blog/[slug]` | Single article (single column centered) | 3600 |
| `/blog/categorii/[slug]` | Category-filtered listing (SSR, SEO-friendly) | 3600 |
| `/pagini/[slug]` | Static pages from Payload (About, legal) | 3600 |

### Blog listing (`/blog`)

- Grid 3 columns (desktop), 2 columns (tablet), 1 column (mobile)
- Category filter tabs above grid (Toate / Ghiduri / Noutăți / Produse în focus)
- Filter is client-side (all posts fetched at build/revalidate, filtered in browser — no extra API call on tab switch)
- Each card: cover image, category badge, title, excerpt, author, read time, date

### Blog post (`/blog/[slug]`)

- Single column centered layout (max ~700px width)
- Structure: category breadcrumb → title → author + date + read time → cover image → Lexical rich text content
- Below article: "Articole similare" (same category, max 3, exclude current)
- `generateStaticParams` for all published posts at build time

### Category page (`/blog/categorii/[slug]`)

- Same grid layout as `/blog` but pre-filtered by category server-side
- `generateStaticParams` for all categories

### Static pages (`/pagini/[slug]`)

- Minimal layout: title + Lexical rich text content
- `generateStaticParams` for all pages with `showInFooter: true` + any others

### Navigation updates

**Header** — add `Blog` link in main nav alongside `Produse` and `Categorii`.

**Footer** — add `Informații` column with links to pages where `showInFooter: true`:
- Despre noi
- GDPR
- Termeni și condiții
- Politica de retur

Footer links fetched at build time from Payload `pages` collection filtered by `showInFooter: true`.

### Data fetching

Storefront fetches from Payload REST API (`http://localhost:3001/api`). Pattern consistent with existing Medusa fetches:

```ts
// lib/cms/client.ts — new
const CMS_URL = process.env.PAYLOAD_URL ?? "http://localhost:3001"

export async function getPosts(categorySlug?: string): Promise<Post[]>
export async function getPost(slug: string): Promise<Post | null>
export async function getCategories(): Promise<Category[]>
export async function getPage(slug: string): Promise<Page | null>
export async function getFooterPages(): Promise<Page[]>
```

- All functions use `fetch()` with `{ next: { revalidate: 3600 } }`
- Only `published` posts returned to storefront (filtered in API query: `?where[status][equals]=published`)

### On-demand revalidation

Payload triggers `POST /api/revalidate?secret=REVALIDATE_SECRET` on publish. `app/api/revalidate/route.ts` (already exists) calls `revalidatePath("/blog")` and `revalidatePath("/pagini")`.

Storefront env var: `REVALIDATE_SECRET=<shared secret>`.
Payload webhook configured in `payload.config.ts` afterChange hook on `posts` and `pages` collections.

## Section 3: Cookie Consent Banner

### Implementation

`vanilla-cookieconsent` v3 — zero React dependencies, 4kb, full GDPR.

Initialized as a `"use client"` component `<CookieConsent />` mounted once in `app/layout.tsx`.

### Categories

| Category | Key | Default | Description |
|---|---|---|---|
| Necesare | `necessary` | Always on | Session, cart, auth cookies |
| Analitice | `analytics` | Off | Future: traffic analytics |
| Marketing | `marketing` | Off | Future: ad retargeting |

### UX

- **Position:** Full-width bar at bottom of viewport (does not block content)
- **First visit:** Bar appears with two buttons: "Acceptă toate" + "Personalizează"
- **"Personalizează"** expands panel with category toggles (Necesare always locked on)
- **Preference stored** in cookie `cc_cookie` (1-year expiry)
- **Re-consent** triggered after 365 days or if categories change
- **Language:** Romanian throughout

### Component structure

```
components/cookie-consent/
└── cookie-consent.tsx    ← "use client", initializes vanilla-cookieconsent
```

## Section 5: Testing

- No unit tests for Payload collections (CMS content, not application logic)
- No unit tests for cookie consent (browser-only library)
- Smoke tests (manual, against running server):
  - Publish post in Payload → verify appears on `/blog`
  - Create legal page → verify renders at `/pagini/[slug]`
  - Cookie consent banner appears on first visit, preference persists on reload

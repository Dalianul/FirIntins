## Section 1: Returns / RMA

### Architecture

Medusa v2's native Returns module handles the data model and lifecycle. A new custom workflow `request-return` in `apps/backend/src/workflows/request-return.ts` orchestrates validation + return creation + email. A new storefront API route on the Medusa backend handles the customer-facing request.

**Files created/modified:**

| File | Change |
|------|--------|
| `apps/backend/src/api/store/orders/[id]/return/route.ts` | New — POST endpoint, validates and runs workflow |
| `apps/backend/src/workflows/request-return.ts` | New — validates, calls initiateReturnWorkflow, sends email |
| `apps/storefront/actions/return.ts` | New — server action calling the Medusa return endpoint |
| `apps/storefront/app/(main)/cont/comenzi/[id]/retur/page.tsx` | New — return request form page |
| `apps/storefront/components/account/return-form.tsx` | New — client form component |
| `apps/storefront/components/account/order-detail.tsx` | Modified — add return button + return status badge |

### Backend: API Route

`POST /store/orders/:id/return`

Request body:
```ts
{
  items: Array<{ line_item_id: string; quantity: number; reason: string }>
  reason: string  // overall return reason
}
```

Validation (before running workflow):
- Authenticated customer owns the order (`order.customer_id === customer.id`)
- Order status is `completed`
- Order `created_at` is within the last 14 days
- Order has no existing return in non-cancelled status

Returns `201` with the created return object, or `400` with a Romanian error message.

### Backend: Workflow `request-return`

```ts
// apps/backend/src/workflows/request-return.ts
import { createWorkflow, createStep, StepResponse } from "@medusajs/framework/workflows-sdk"
import { initiateReturnWorkflow } from "@medusajs/medusa/core-flows"

const createReturnStep = createStep(
  "create-return-step",
  async ({ orderId, items }, { container }) => {
    const result = await initiateReturnWorkflow(container).run({
      input: {
        order_id: orderId,
        items,
      },
    })
    return new StepResponse(result)
  }
)

const sendReturnConfirmationStep = createStep(
  "send-return-confirmation-step",
  async ({ order, customerEmail, returnItems }, { container }) => {
    const notificationService = container.resolve("notification")
    await notificationService.send({
      to: customerEmail,
      channel: "email",
      template: "return-confirmation",
      data: {
        order_id: order.id,
        items: returnItems,
        return_address: process.env.RETURN_ADDRESS,
      },
    })
    return new StepResponse({})
  }
)

export const requestReturnWorkflow = createWorkflow(
  "request-return",
  (input: { orderId: string; items: ReturnItem[]; order: Order; customerEmail: string }) => {
    const returnResult = createReturnStep({ orderId: input.orderId, items: input.items })
    sendReturnConfirmationStep({
      order: input.order,
      customerEmail: input.customerEmail,
      returnItems: input.items,
    })
    return returnResult
  }
)
```

### Backend: SendGrid Email Template

New env var in `apps/backend/.env`: `RETURN_ADDRESS=<full return address string>`

### Frontend: Return Form Page

`app/(main)/cont/comenzi/[id]/retur/page.tsx` — Server Component:
- Fetches order via `getCustomerOrder(id)` (authenticated)
- If order not within 14 days or not `completed` status → redirect to `/cont/comenzi/[id]`
- Passes order items to `<ReturnForm order={order} />`

`components/account/return-form.tsx` — Client Component:
```tsx
"use client"
// Props: order: Order
// State: selectedItems (Map<line_item_id, quantity>), reason, submitting, error
// Fields:
//   - Checkbox list of order items with quantity spinners
//   - Reason select: "Produs deteriorat", "Produs greșit", "M-am răzgândit", "Altul"
//   - Submit button: "Trimite cererea de retur"
// On submit: calls requestReturn server action
// On success: router.push(`/cont/comenzi/${order.id}?retur=success`)
```

`actions/return.ts` — Server Action:
```ts
"use server"
export async function requestReturn(orderId: string, data: ReturnFormData) {
  const customer = await getCustomer()
  if (!customer) redirect("/login")

  const response = await medusa.client.fetch(
    `/store/orders/${orderId}/return`,
    { method: "POST", body: data, headers: { Authorization: `Bearer ${token}` } }
  )

  if (!response.ok) {
    return { success: false, error: await response.json() }
  }
  return { success: true }
}
```

## Section 2: Tracking Number Display

### Architecture

No backend changes. Medusa stores `tracking_links: [{ url, tracking_number }]` on each fulfillment. The storefront query needs to expand fulfillments, then the order detail renders the tracking data.

**Files modified:**

| File | Change |
|------|--------|
| `apps/storefront/lib/medusa/queries.ts` | Modified — expand `fulfillments.tracking_links` in `getCustomerOrder` |
| `apps/storefront/components/account/order-detail.tsx` | Modified — add tracking block |

### Query Change

In `getCustomerOrder` (or equivalent in `queries.ts`), add to the fields/expand:
```ts
"fulfillments.tracking_links"
// or in the fields array: "+fulfillments.tracking_links.url", "+fulfillments.tracking_links.tracking_number"
```

### Display

Below the shipping address in `order-detail.tsx`, add:

```tsx
{order.fulfillments?.some(f => f.tracking_links?.length > 0) && (
  <div>
    <h4>Urmărire colet</h4>
    {order.fulfillments
      .flatMap(f => f.tracking_links ?? [])
      .map((link, i) => (
        <a key={i} href={link.url} target="_blank" rel="noopener noreferrer">
          {link.tracking_number}
        </a>
      ))
    }
  </div>
)}
```

Only rendered when tracking links exist. No empty state needed.

## Section 3: Newsletter Welcome Email + Discount

### Architecture

Payload CMS stores subscribers (deduplication). The storefront API route sends email directly via SendGrid. A shared Medusa discount code is referenced via env var.

**Files created/modified:**

| File | Change |
|------|--------|
| `apps/storefront/collections/NewsletterSubscribers.ts` | New — Payload collection |
| `apps/storefront/payload.config.ts` | Modified — register NewsletterSubscribers collection |
| `apps/storefront/app/api/newsletter/route.ts` | Modified — add Payload write + SendGrid send |

### Payload Collection

```ts
// collections/NewsletterSubscribers.ts
import type { CollectionConfig } from "payload"

export const NewsletterSubscribers: CollectionConfig = {
  slug: "newsletter-subscribers",
  admin: { useAsTitle: "email" },
  access: { read: () => false, create: () => false, update: () => false, delete: () => false },
  fields: [
    { name: "email", type: "email", required: true, unique: true },
    { name: "subscribedAt", type: "date", defaultValue: () => new Date().toISOString() },
  ],
}
```

`access: read/create/update/delete` all `() => false` to prevent public API access — writes happen server-side only via the local Payload API.

### API Route

`app/api/newsletter/route.ts`:

```ts
import { getPayload } from "payload"
import config from "@payload-config"
import sgMail from "@sendgrid/mail"

export async function POST(req: Request) {
  const { email } = await req.json()

  // Validate email format
  if (!email || !/^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(email)) {
    return Response.json({ error: "Email invalid" }, { status: 400 })
  }

  const payload = await getPayload({ config })

  // Check for existing subscriber — silent success (no duplicate email)
  const existing = await payload.find({
    collection: "newsletter-subscribers",
    where: { email: { equals: email } },
    limit: 1,
  })
  if (existing.totalDocs > 0) {
    return Response.json({ success: true })
  }

  // Create subscriber record
  await payload.create({
    collection: "newsletter-subscribers",
    data: { email, subscribedAt: new Date().toISOString() },
  })

  // Send welcome email via SendGrid
  sgMail.setApiKey(process.env.SENDGRID_API_KEY!)
  await sgMail.send({
    to: email,
    from: process.env.SENDGRID_FROM_EMAIL!,
    subject: `Bun venit — codul tău de reducere ${process.env.NEXT_PUBLIC_NEWSLETTER_DISCOUNT_CODE}`,
    text: [
      "Mulțumim că te-ai abonat!",
      "",
      `Codul tău de reducere: ${process.env.NEXT_PUBLIC_NEWSLETTER_DISCOUNT_CODE}`,
      "Reducere 10% la prima comandă.",
      "",
      "Folosește codul la finalizarea comenzii.",
      "",
      "Pescuit plăcut,",
      "Echipa FirIntins",
    ].join("\n"),
  })

  return Response.json({ success: true })
}
```

```

```

### Dependencies

## Section 4: VAT / CUI Display

### Architecture

CUI (Romanian company tax ID) is an optional field added to the checkout address step. It's saved to `order.metadata.cui`. Displayed on order confirmation and account order detail. The existing TVA lines are audited in all three checkout/post-purchase locations.

**Files modified:**

| File | Change |
|------|--------|
| `apps/storefront/components/checkout/address-step.tsx` | Modified — add optional CUI field |
| `apps/storefront/lib/schema/checkout.ts` | Modified — add optional `cui` to address schema |
| `apps/storefront/actions/checkout.ts` | Modified — pass `cui` to order metadata |
| `apps/storefront/app/(main)/(checkout)/checkout/confirmare/[orderId]/page.tsx` | Modified — show CUI if present |
| `apps/storefront/components/account/order-detail.tsx` | Modified — show CUI + audit TVA line |

### Checkout Form

In the address step form, add after the company/address fields:

```tsx
<div>
  <Label htmlFor="cui">Cod fiscal (CUI) — opțional</Label>
  <Input id="cui" {...register("cui")} placeholder="RO12345678" />
</div>
```

Zod schema addition:
```ts
cui: z.string().optional(),
```

### Server Action

In the checkout server action that creates/updates the order, pass CUI to metadata:

```ts
await medusa.orders.update(orderId, {
  metadata: { cui: data.cui ?? null },
})
```

Or if CUI is collected at address submission time, include in the address update call.

### CUI Display

On both order confirmation and account order detail, after the address block:

```tsx
{order.metadata?.cui && (
  <p className="text-sm text-fog">Cod fiscal: {order.metadata.cui}</p>
)}
```

### TVA Audit

Verify these three locations all render the TVA line correctly:

1. **`components/checkout/order-summary.tsx`** — `cart.tax_total / 100` formatted as RON
2. **`app/(main)/(checkout)/checkout/confirmare/[orderId]/page.tsx`** — `order.tax_total / 100`
3. **`components/account/order-detail.tsx`** — `order.tax_total / 100`

If any are missing, add the line:
```tsx
<div className="flex justify-between">
  <span>TVA</span>
  <span>{formatPrice(order.tax_total)}</span>
</div>
```

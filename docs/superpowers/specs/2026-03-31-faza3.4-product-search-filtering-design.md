# Faza 3.4 — Product Search & Filtering: Design Spec

**Date:** 2026-03-31
**Project:** FirIntins — fishing equipment e-commerce (Medusa v2 + Next.js 16)
**Status:** Approved

---

## Overview

Add full-text product search and extended filtering to the storefront. Search is available globally via the header and redirects to the unified product listing page `/produse`, which handles both browsing and search results. All filter state lives in URL query params.

---

## Scope

**In Faza 3.4:**
- Global header search (inline expand → redirect to `/produse?q=...`)
- Stock availability toggle (in stock / out of stock)
- Sort order dropdown (relevance, price asc/desc, newest, title A–Z)
- Extend `getProducts` with `q`, `order`, `in_stock` params
- Suspense skeleton for product grid during filter/search navigation

**Out of scope (future phases):**
- MeiliSearch or any external search engine
- Autocomplete / live search suggestions
- Rating-based filtering
- Attribute/tag-based filtering (brand, material, etc.)
- Search history or recent searches

## Architecture

### URL Schema

All filter and search state is encoded in URL search params:

```
/produse?q=nălucă&category=spinning&price_min=50&price_max=300&in_stock=true&sort=price_asc
```

| Param | Type | Values |
|-------|------|--------|
| `q` | string | free text search query |
| `category` | string | category slug |
| `price_min` | number | minimum price in RON |
| `price_max` | number | maximum price in RON |
| `in_stock` | boolean | `"true"` when active, absent otherwise |
| `sort` | string | `relevance` \| `price_asc` \| `price_desc` \| `newest` \| `title_asc` |

### Data Flow

1. User types in header search → Enter → `router.push('/produse?q=' + encodeURIComponent(value))`
2. `/produse/page.tsx` (Server Component) receives `searchParams`
3. Passes params to `<ProductGrid searchParams={...} />` (async Server Component, wrapped in `<Suspense>`)
4. `ProductGrid` calls `getProducts(params)` and renders product cards
5. Filter Client Components read current values from `searchParams` (as props) and call `router.push` with updated params on change

### URL Update Helper

Each filter Client Component uses this pattern:

```ts
function updateSearchParam(key: string, value: string | null) {
  const params = new URLSearchParams(searchParams.toString())
  if (value === null) params.delete(key)
  else params.set(key, value)
  router.push('/produse?' + params.toString())
}
```

## Components

### New Components

#### `components/layout/search-button.tsx`
- **Type:** Client Component
- **Closed state:** icon button (`Search` from Lucide) in header, alongside wishlist and cart icons
- **Open state:** full-width input in header, replacing nav links; `X` button at right; `autoFocus` on open
- **Transition:** `motion/react` `AnimatePresence` — input slides in, nav fades out
- **Submit:** Enter key → `router.push('/produse?q=' + encodeURIComponent(value))`; empty query → `router.push('/produse')`
- **Dismiss:** Escape key or `X` button → close and clear input
- **Pre-fill:** initializes with current `useSearchParams().get('q')` value when on `/produse`
- **Mobile:** same behavior, input fills available header width

#### `components/product/product-grid.tsx`
- **Type:** async Server Component
- Accepts `searchParams` as props
- Calls `await getProducts(mappedParams)`
- Renders product cards (reuses existing card component)
- Rendered inside `<Suspense fallback={<ProductGridSkeleton />}>` in the page

#### `components/product/product-grid-skeleton.tsx`
- 6–8 placeholder cards with `animate-pulse`
- Same dimensions as real product cards

#### `components/product/in-stock-toggle.tsx`
- **Type:** Client Component
- Toggle UI (shadcn Switch or custom)
- `on` → sets `?in_stock=true`; `off` → removes `in_stock` param
- Reads current value from `searchParams.in_stock` prop

#### `components/product/sort-select.tsx`
- **Type:** Client Component
- Dropdown (shadcn Select or native `<select>`)
- Options: Relevanță (default), Preț crescător, Preț descrescător, Cele mai noi, Titlu A–Z
- Reads current value from `searchParams.sort` prop; writes `?sort=<value>` on change
- Aligned to the right of the filter bar (`ml-auto`)

### Modified Components

#### `app/(main)/(shop)/produse/page.tsx`
- Extract product fetching + grid rendering into `<ProductGrid>`
- Wrap `<ProductGrid>` in `<Suspense fallback={<ProductGridSkeleton />}>`
- Pass `searchParams` as props to filter components and `ProductGrid`
- Add "Resetează filtrele" button when any filter is active

#### `lib/medusa/queries.ts`
- Add to `ProductParams` interface: `q?: string`, `order?: string`
- Map `sort` param to Medusa `order` format:

```ts
const ORDER_MAP: Record<string, string> = {
  price_asc:  'variants.calculated_price.calculated_amount ASC',
  price_desc: 'variants.calculated_price.calculated_amount DESC',

}
```

- **`in_stock` filter:** attempt `variants.inventory_quantity[gt]=0` via raw param. If Medusa JS SDK does not expose this directly, filter post-fetch from returned products. This is acceptable for Faza 3.4 given typical catalog size.

#### `components/product/price-filter.tsx`

#### `components/product/category-filter.tsx`

## Filter Bar Layout

Horizontal bar above the product grid:

```
[ Categorie ▾ ]  [ Preț: 50–300 RON ▾ ]  [ În stoc ○ ]          [ Sortează: Relevanță ▾ ]
```

- All filters left-aligned, sort right-aligned (`justify-between` or `ml-auto`)
- On mobile: filters wrap to second line or collapse into a "Filtre" drawer button (implementation decision at build time)
- "Resetează filtrele" link appears below/beside the bar when any non-default filter is active

| `newest` | `created_at DESC` |

## What Is Explicitly Not Built

- No dedicated `/cautare` route — `/produse` is the unified search + browse page
- No MeiliSearch, Algolia, or any external search service
- No live autocomplete or search-as-you-type
- No search analytics
- No attribute or tag filters
- No rating filter

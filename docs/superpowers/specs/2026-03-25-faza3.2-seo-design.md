## Secțiunea 1 — Sitemap + Robots

### `app/sitemap.ts`

Funcție async `sitemap()` ce returnează `MetadataRoute.Sitemap`. Combină 4 surse de date:

| Sursă | URL-uri | `changeFrequency` | `priority` |
|---|---|---|---|
| Hardcodat static | `/`, `/produse`, `/blog` | `weekly` / `daily` / `weekly` | `1.0`, `0.9`, `0.7` |
| Medusa — produse | `/produse/[handle]` | `weekly` | `0.8` |
| Medusa — categorii | `/categorii/[slug]` | `monthly` | `0.6` |
| Payload — posts publicate | `/blog/[slug]` | `weekly` | `0.7` |
| Payload — pages (`showInFooter: true`) | `/pagini/[slug]` | `monthly` | `0.5` |

**Resilience:** fiecare sursă externă (Medusa, Payload) este wrapped în `try/catch`. Dacă o sursă eșuează, celelalte continuă — sitemap-ul returnează URL-urile disponibile, nu aruncă eroare.

**`lastModified`:** setat din `updatedAt` pentru produse/posts/pages unde există; `new Date()` pentru pagini statice.

**URL base:** `process.env.NEXT_PUBLIC_SERVER_URL` (ex: `https://firintins.ro`) — nu hardcodat.

**Date fetching:** `getCachedProducts()` din `lib/medusa/queries.ts` (deja există), `getCachedCategories()` (deja există), `getCachedPosts()` și `getCachedPage` din `lib/cms/client.ts` (deja există). Sitemap-ul refolosește funcțiile existente.

### `app/robots.ts`

Funcție `robots()` ce returnează `MetadataRoute.Robots`:

```
User-Agent: *
Allow: /
Disallow: /checkout
Disallow: /cont
Disallow: /api
Disallow: /admin
Sitemap: {NEXT_PUBLIC_SERVER_URL}/sitemap.xml
```

## Secțiunea 3 — OG Images dinamice

### Strategie în 2 niveluri

**Nivel 1 — Imagine specifică per rută** (colocată cu `page.tsx`):
- `app/(main)/(shop)/produse/[handle]/opengraph-image.tsx`
- `app/(main)/(shop)/blog/[slug]/opengraph-image.tsx`

**Nivel 2 — Fallback static:**
- `public/og-default.jpg` — 1200×630px, imaginea de brand FirIntins

### Implementare `opengraph-image.tsx`

Folosește `ImageResponse` din `next/og` (Edge Runtime, built-in Next.js, zero dependențe externe). Generează PNG 1200×630px server-side.

**Layout OG image produs (`/produse/[handle]`):**
- Background: thumbnail-ul produsului (dacă există) sau `--color-bg` (#0f0e0b)
- Overlay gradient dark pe jumătatea inferioară
- Text: titlul produsului (font-size mare, alb), "FirIntins" mic jos-dreapta
- Fallback: dacă produsul nu există → redirect la imaginea statică default

**Layout OG image blog post (`/blog/[slug]`):**
- Background: cover image-ul postului (dacă există) sau `--color-bg`
- Overlay gradient dark pe jumătatea inferioară
- Text: titlul postului + categoria (badge verde `--color-moss`), "FirIntins Blog" mic jos-dreapta
- Fallback: dacă postul nu există → imaginea statică default

**Font în ImageResponse:** `Cormorant Garamond` — fetchat din Google Fonts în funcția `opengraph-image.tsx` via `fetch()` (pattern standard Next.js).

**Runtime:** `export const runtime = "edge"` pe ambele fișiere `opengraph-image.tsx`.

**`params` tip:** `Promise<{ handle: string }>` respectiv `Promise<{ slug: string }>` — await obligatoriu (Next.js 16).

## Secțiunea 4 — Error Pages

Trei fișiere în `app/` — UI consistent cu design system-ul (dark background `--color-bg`, Cormorant headings, CSS variables existente, fără layout-ul normal al site-ului pentru `global-error`).

### `app/not-found.tsx`

Server Component. Afișat la orice rută inexistentă (404).

Conținut:
- Heading `404` mic, suprascris de "Pagina nu a fost găsită" (Cormorant)
- Subtitlu explicativ scurt în română
- Două butoane: "Înapoi la homepage" (`/`) + "Vezi produsele" (`/produse`)
- Folosește layout-ul normal `(main)` — header + footer prezente

### `app/error.tsx`

Client Component (`"use client"`). Primeşte `{ error, reset }` props. Afișat la erori runtime în orice segment.

Conținut:
- "Ceva nu a funcționat" (Cormorant heading)
- Subtitlu generic, fără stack trace
- Buton "Încearcă din nou" → apelează `reset()`
- Buton secundar "Înapoi la homepage"
- `useEffect` → `console.error(error)` pentru logging dev

### `app/global-error.tsx`

Client Component (`"use client"`). Fallback pentru erori la nivel de root layout. Renderează propriul `<html><body>` (layout-ul normal nu e disponibil).

Conținut:
- `<html lang="ro"><body>` cu background `#0f0e0b` inline
- Mesaj minimal: "Eroare critică. Reîncarcă pagina."
- Buton `onClick={() => window.location.reload()}`

## Pattern-uri Next.js 16 obligatorii

**Params în route dinamice:** Next.js 16 returnează `params` ca `Promise`. Toate componentele de pagină și funcțiile `generateMetadata` cu rute dinamice trebuie să primească `params: Promise<{ [param]: string }>` și să facă `await params` înainte de a accesa proprietățile:

```ts
// CORECT — Next.js 16
type Props = { params: Promise<{ handle: string }> }

export async function generateMetadata({ params }: Props) {
  const { handle } = await params
  // ...
}

export default async function Page({ params }: Props) {
  const { handle } = await params
  // ...
}
```

Același pattern se aplică la `opengraph-image.tsx`.

**`BASE_URL` constant:** Creează `lib/constants.ts` cu:

```ts
export const BASE_URL = process.env.NEXT_PUBLIC_SERVER_URL ?? "https://firintins.ro"
```

Toate `generateMetadata` și `robots.ts` importă `BASE_URL` din acest fișier — nu se repetă `process.env.NEXT_PUBLIC_SERVER_URL` în fiecare modul.

## Secțiunea 1 — Sitemap + Robots

### `app/sitemap.ts`

Funcție async `sitemap()` ce returnează `MetadataRoute.Sitemap`. Combină 4 surse de date.

**Date fetching:** Sitemap-ul folosește **direct funcțiile non-cached** din `lib/medusa/queries.ts` și `lib/cms/client.ts` — nu sunt necesare wrapped-uri cached deoarece Next.js cachează output-ul întregului `sitemap.ts` la nivel de framework. Funcțiile folosite:
- `getProducts()` din `lib/medusa/queries.ts` (deja există — fără wrapper cached pentru Medusa)
- `getCategories()` din `lib/medusa/queries.ts` (deja există)
- `getPosts()` din `lib/cms/client.ts` (deja există — NU `getCachedPosts()`)
- `getFooterPages()` din `lib/cms/client.ts` (deja există — returnează pages cu `showInFooter: true`; NU `getCachedFooterPages()`)

**Notă:** `lib/cms/client.ts` exportă atât funcții raw (`getPosts`, `getCategories` etc.) cât și cached wrappers (`getCachedPosts`, `getCachedCategories` etc.). Sitemap-ul folosește exclusiv variantele raw. Paginile de conținut (page components și `generateMetadata`) folosesc variantele cached.

| Sursă | URL-uri | `changeFrequency` | `priority` |
|---|---|---|---|
| Hardcodat static | `/`, `/produse`, `/blog` | `weekly` / `daily` / `weekly` | `1.0`, `0.9`, `0.7` |
| Medusa — produse | `/produse/[handle]` | `weekly` | `0.8` |
| Medusa — categorii | `/categorii/[slug]` | `monthly` | `0.6` |
| Payload — posts publicate | `/blog/[slug]` | `weekly` | `0.7` |
| Payload — pages (`showInFooter: true`) | `/pagini/[slug]` | `monthly` | `0.5` |

**Resilience:** fiecare sursă externă (Medusa, Payload) este wrapped în `try/catch` individual. Dacă o sursă eșuează, celelalte continuă — sitemap-ul returnează URL-urile disponibile, nu aruncă eroare.

**`lastModified`:**
- Produse/posts/pages: setat din câmpul `updatedAt` al documentului — dacă `updatedAt` e `null` sau `undefined`, fallback la `new Date("2026-03-25")`
- Pagini statice hardcodate (`/`, `/produse`, `/blog`): valoare fixă `new Date("2026-03-25")` — **nu** `new Date()` (care se regenerează la fiecare request și semnalează fals "conținut nou" crawlerelor)

**URL base:** importat din `lib/constants.ts` → `BASE_URL`

**Revalidare:** `export const revalidate = 3600` la nivel de modul — sitemap-ul se regenerează la fiecare oră.

### `app/robots.ts`

Funcție `robots()` ce returnează `MetadataRoute.Robots`. Importă `BASE_URL` din `lib/constants.ts`.

```
User-Agent: *
Allow: /
Disallow: /checkout
Disallow: /cont
Disallow: /api       ← Payload CMS REST/GraphQL intern — nu trebuie indexat
```

## Secțiunea 2 — `generateMetadata` per pagină

Paginile care au deja `generateMetadata` complet rămân neatinse, cu excepția `/blog/[slug]` unde se adaugă `canonical`.

Toate paginile importă `BASE_URL` din `lib/constants.ts`.

| Pagină | Title pattern | Description | Canonical | OG Image |
|---|---|---|---|---|
| `/` | `FirIntins — Echipamente pescuit la crap` | Tagline brand fix | `${BASE_URL}/` | fallback static |
| `/produse` | `Produse — FirIntins` | Text generic fix | `${BASE_URL}/produse` | fallback static |
| `/produse/[handle]` | `${product.title} — FirIntins` | `product.description` (trim 160 chars) | `${BASE_URL}/produse/${handle}` | thumbnail produs via `opengraph-image.tsx` |
| `/categorii/[slug]` | `${category.name} — FirIntins` | Text generic | `${BASE_URL}/categorii/${slug}` | fallback static |
| `/blog` | `Blog — FirIntins` | Text generic fix | `${BASE_URL}/blog` | fallback static |
| `/blog/[slug]` | deja există — **adăugăm `canonical`** | deja există | `${BASE_URL}/blog/${slug}` | cover image post via `opengraph-image.tsx` — **nu setăm `openGraph.images` explicit** (Next.js îl preia automat din fișierul colocat) |
| `/blog/categorii/[slug]` | deja există — **adăugăm `canonical` + `openGraph.images`** | deja există | `${BASE_URL}/blog/categorii/${slug}` | fallback static |
| `/pagini/[slug]` | `${page.title} — FirIntins` | Primele 160 chars text din `page.content` (text extras din Lexical JSON — vezi algoritm mai jos) | `${BASE_URL}/pagini/${slug}` | fallback static |
| `/login`, `/register` | Titluri simple | Nu are | Nu are | skip — adăugăm `robots: { index: false }` |

**Extragere text din Lexical JSON** (folosit pentru description la `/pagini/[slug]`):

```ts
function extractTextFromLexical(content: unknown): string {
  if (!content || typeof content !== "object") return ""
  const node = content as Record<string, unknown>
  if (node.type === "text" && typeof node.text === "string") return node.text
  if (Array.isArray(node.children)) {
    return (node.children as unknown[]).map(extractTextFromLexical).join(" ")
  }
  if (node.root) return extractTextFromLexical(node.root)
  return ""
}
// Usage: extractTextFromLexical(page.content).slice(0, 160).trim()
```

**`alternates.canonical`:** setat pe fiecare pagină pentru a evita conținut duplicat (ex: query params de filtrare).

**`openGraph.images`:** pentru paginile fără `opengraph-image.tsx` specific, se setează explicit `openGraph: { images: [{ url: "${BASE_URL}/og-default.jpg" }] }`.

## Secțiunea 3 — OG Images dinamice

### Strategie în 2 niveluri

**Nivel 1 — Imagine specifică per rută** (colocată cu `page.tsx`):
- `app/(main)/(shop)/produse/[handle]/opengraph-image.tsx`
- `app/(main)/(shop)/blog/[slug]/opengraph-image.tsx`

**Nivel 2 — Fallback static:**
- `public/og-default.jpg` — 1200×630px, imaginea de brand FirIntins (asset adăugat manual)

### Implementare `opengraph-image.tsx`

Folosește `ImageResponse` din `next/og` (Edge Runtime, built-in Next.js 16, zero dependențe externe). Generează PNG 1200×630px server-side.

**Runtime:** `export const runtime = "edge"` pe ambele fișiere. Edge Runtime nu are acces la sistemul de fișiere (`fs`) — fallback-ul nu poate citi `public/og-default.jpg` de pe disk.

**Fallback comportament (IMPORTANT):** Dacă produsul/postul nu există sau nu are imagine, se randează un `ImageResponse` minimal cu brand colors — **nu** `notFound()`, **nu** `redirect()`, **nu** `fs.readFileSync()`:

```tsx
// Fallback — brand colors, fără imagine externă
return new ImageResponse(
  <div style={{ background: "#0f0e0b", width: "100%", height: "100%", display: "flex",
    alignItems: "center", justifyContent: "center" }}>
    <span style={{ color: "#faf8f3", fontSize: 48, fontFamily: "serif" }}>FirIntins</span>
  </div>,
  { width: 1200, height: 630 }
)
```

**Layout OG image produs (`/produse/[handle]`):**
- Fetch produs via `getProduct(handle)` direct (non-cached, Edge Runtime)
- Dacă `product?.thumbnail` există: imagine ca background (style `objectFit: cover`)
- Overlay gradient dark pe jumătatea inferioară (`background: linear-gradient(transparent, rgba(0,0,0,0.8))`)
- Text: titlul produsului (font-size 48px, alb `#faf8f3`), "FirIntins" 20px jos-dreapta
- Dacă produsul nu există sau nu are thumbnail: fallback brand colors de mai sus

**Layout OG image blog post (`/blog/[slug]`):**
- Fetch post via `getPost(slug)` direct (non-cached, Edge Runtime)
- Dacă `post?.coverImage?.url` există: imagine ca background
- Overlay gradient dark pe jumătatea inferioară
- Text: titlul postului (48px, alb), badge categorie verde (`#4a5e3a`, text alb, 20px), "FirIntins Blog" 20px jos-dreapta
- Dacă postul nu există sau nu are cover image: fallback brand colors

**Font în ImageResponse:** Cormorant Garamond fetchat din Google Fonts:
```ts
const font = await fetch("https://fonts.gstatic.com/s/cormorantgaramond/...").then(r => r.arrayBuffer())
// inclus în options: { fonts: [{ name: "Cormorant", data: font, style: "normal" }] }
```
Dacă fetch-ul fontului eșuează (edge timeout), continuă cu `font-family: serif` — nu aruncă eroare.

**`params` tip:** `Promise<{ handle: string }>` respectiv `Promise<{ slug: string }>` — `await params` obligatoriu (Next.js 16 pattern).

**Imagine produs URL:** Medusa returnează URL-uri absolute (ex: `http://localhost:9000/uploads/...`). În `ImageResponse`, imaginile externe trebuie să fie URL-uri accesibile din Edge Runtime — în producție funcționează. În dev, dacă URL-ul e localhost, fallback la brand colors.

## Secțiunea 4 — Error Pages

Trei fișiere în `app/` — UI consistent cu design system-ul, CSS variables disponibile via inline styles (nu Tailwind, pentru că `global.css` nu e importat la nivel global disponibil în error boundaries).

**Notă tehnică:** `app/error.tsx` și `app/global-error.tsx` sunt Client Components și rulează înainte ca layout-urile să fie disponibile garantat. Folosiți inline styles cu valorile hardcodate din design system pentru robustețe (`background: "#0f0e0b"`, `color: "#faf8f3"` etc.).

### `app/(main)/not-found.tsx`

Server Component. Afișat la orice rută inexistentă din grupul `(main)` (statuscode 404).

**Plasare:** `app/(main)/not-found.tsx` (nu `app/not-found.tsx`) — astfel moștenește layout-ul `(main)` care include Header + Footer. Rutele din `(payload)` (`/admin`) au propriul handling.

Conținut:
- Heading Cormorant: "Pagina nu a fost găsită"
- Subtitlu explicativ scurt în română
- Două butoane: "Înapoi la homepage" (`/`) + "Vezi produsele" (`/produse`) — `<Link>` standard

Accesibilitate: semantic HTML (`<main>`, `<h1>`, `<a>` nu `<button>` pentru navigare).

### `app/error.tsx`

Client Component (`"use client"`). Primește `{ error: Error & { digest?: string }, reset: () => void }`.

Conținut:
- `useEffect(() => { console.error(error) }, [error])` — logging dev
- Heading: "Ceva nu a funcționat"
- Subtitlu generic în română, fără stack trace în UI
- Buton primar "Încearcă din nou" → `onClick={reset}`, `autoFocus` pentru keyboard navigation
- Buton/link secundar "Înapoi la homepage" → `<Link href="/">`

### `app/global-error.tsx`

Client Component (`"use client"`). Fallback pentru erori la nivel de root layout. Trebuie să randeze propriul `<html><body>`.

Conținut:
- `<html lang="ro"><body style={{ background: "#0f0e0b", color: "#faf8f3", ... }}>`
- Mesaj minimal: "Eroare critică. Te rugăm să reîncarci pagina."
- `<button onClick={() => reset()}>Reîncearcă</button>` — nu `window.location.reload()` (preferăm `reset()` prop)
